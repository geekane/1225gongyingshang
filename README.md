# 创建管理系统 (SRM 2.0)

本项目基于 RuoYi-Vue3 进行深度定制改造，旨在打造一个高效、灵活的供应商自主注册与全生命周期管理系统。

## 核心架构：双门户设计

项目实现了“双门户”管理逻辑，通过角色自动切换业务视角：
- **管理端（采购端）**：供内部人员进行供应商入驻审核、档案管理及系统运维。
- **供应商门户**：供合作伙伴完成自主注册、资料完善及后续业务协作。

## 已完成的改造特性

1.  **分步自主注册**：全新的供应商注册流程，包含准入选择、协议签署、身份核验、账号激活四个阶段。
2.  **角色菜单隔离**：
    - 管理员登录（`admin`）展示完整看板与管理菜单。
    - 供应商登录（`supplier_test`）仅展示资料完善等专有页面。
3.  **看板化首页**：仿照现代化 SRM 设计，重构首页为实时数据看板（访客、订单、金额统计及趋势图）。
4.  **轻量化后端**：
    - 适配了 Cloudflare Worker + D1 数据库的后端方案（位于 `api/` 目录）。
    - 实现了兼容的原生登录、路由获取及注销接口。
5.  **品牌深度定制**：全局去“若依”化，统一更名为“创建管理系统”，清理了冗余的外部链接与图标。

## 快速开始

### 前端运行
```bash
# 安装依赖
pnpm install

# 启动服务
pnpm dev
```

### 后端说明 (Cloudflare Worker)
项目后端代码位于 `api/` 目录下，使用 Cloudflare D1 作为数据库。
1.  **初始化数据库**：执行 `api/schema.sql` 中的脚本。
2.  **配置变量**：在 `api/wrangler.toml` 中配置 `JWT_SECRET` 及 `database_id`。
3.  **启动本地后端**：`npx wrangler dev --local`。
